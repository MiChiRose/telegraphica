# Telegraphica v0.4.0-alpha.1 — First Usable Public Alpha

This alpha is intended to replace the earlier `v0.3.0-alpha.*` test releases.
It should be built on the OS X 10.9.5 Mavericks validation machine with Xcode
6.2 and a known-working Mavericks-compatible TDLib JSON library bundled inside
the app.

## Highlights

- Sign in with Telegram phone/code/2FA through TDLib JSON.
- Load chats, supergroup topics, message history, unread state, and read state.
- Send text, multiline messages, photos with preview/caption, reactions, and
  voice messages.
- Preview photos, grouped media, videos, voice messages, and supported document
  metadata.
- Notification Center alerts, Dock unread badges, click-through from
  notifications, muted-chat indicators, and notification settings.
- Profile, Settings, About, diagnostics, themes, language selector, downloads
  folder, and update checks.
- About panel now points to the public GitHub repository.

## Expected Assets

- `Telegraphica-v0.4.0-alpha.1-macos10.9-x86_64.dmg` — HFS+ drag-and-drop
  installer image for Mavericks.
- `Telegraphica-v0.4.0-alpha.1-macos10.9-x86_64.app.zip` — zipped app bundle.
- matching `.sha256` files.

## Build

On the legacy Mac:

```bash
./scripts/package_legacy_release_artifacts.sh --tdjson /path/to/libtdjson.dylib
```

The script rebuilds Telegraphica, validates and bundles TDLib, creates the
Mavericks-mountable installer DMG/app zip, and writes release artifacts into
`dist/`.

## Known Alpha Gaps

- Sticker and GIF rendering is still incomplete and can fall back to emoji.
- Release confidence still comes from OS X 10.9.5 / Xcode 6.2 HITL testing.
- The app is unsigned/not notarized, so Gatekeeper may require right-click Open.
- A public installer is valid only when `libtdjson.dylib` is bundled inside
  `Telegraphica.app/Contents/Frameworks/`.

## Security Notes

Release assets must not include Telegram `api_id`, `api_hash`, phone numbers,
login codes, 2FA passwords, TDLib databases, session files, local Keychain
material, or user media caches.
